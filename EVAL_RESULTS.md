# Eval Results

Official run of `run_eval.py` against the committed 75-record `eval_dataset.json`,
gated by the thresholds in `eval_config.py` (see that file for the rationale).

## Fallback-mode run — 2026-07-07

**Configuration:** no `ANTHROPIC_API_KEY` set (`.env` from `.env.example` as
committed) — every decision produced by `validator.deterministic_fallback()`.
This is the configuration this repo's own Quick Start produces.

```
[Eval] Mode: fallback (0 keyed / 75 fallback / 75 total)
[Eval] Agreement: 68/75
[Eval] Gate (fallback): >= 68
[Eval] Misses (7):
  LEAD-0033: expected=manual_review actual=qualify
  LEAD-0037: expected=manual_review actual=disqualify
  LEAD-0044: expected=manual_review actual=disqualify
  LEAD-0055: expected=hold actual=manual_review
  LEAD-0061: expected=hold actual=manual_review
  LEAD-0063: expected=hold actual=manual_review
  LEAD-0070: expected=hold actual=manual_review
[Eval] RESULT: PASS
```

**Agreement: 68/75 — PASS** against the fallback gate (≥ 68/75).

**The 7 misses are the documented LLM-layer cases.** All seven require
weighing retrieved context or nuance the deterministic rules don't encode —
exactly the gap this system's LLM decision layer exists to close (see
[Known Limitations](README.md#known-limitations) and Outcome bullet 1 in
the README). `deterministic_fallback()` only has four rules (confidence
< 0.35 → disqualify; confidence > 0.70 + high_value → qualify;
support_escalation → escalate; everything else → manual_review), so it
structurally cannot produce `hold` at all, and it cannot distinguish
"genuinely ambiguous, needs manual review" from "confident but wrong"
without reading the retrieved precedent.

## Honest limitations (fallback run)

- **Synthetic data.** `eval_dataset.json` is generated by
  `dataset_generator.py`, not real client data. The 68/75 number describes
  how well four fixed rules match a generated label set, not real-world
  decision accuracy.
- **Small n.** 75 records is not enough to bound the true agreement rate
  tightly; the 68/75 figure is a point estimate, not a statistically
  powered result.

## Keyed run — 2026-07-07

**Configuration:** real `ANTHROPIC_API_KEY` active, `LLM_MODEL = "claude-sonnet-4-6"`
(the originally hardcoded `claude-sonnet-4-20250514` had retired and 404'd on
every request; this is the first successful keyed run).

```
[Eval] Mode: keyed (75 keyed / 0 fallback / 75 total)
[Eval] Agreement: 58/75
[Eval] Gate (keyed): >= 69
[Eval] Keyed target (reported, not gated): 71/75
[Eval] Misses (17):
  LEAD-0015: expected=manual_review actual=hold
  LEAD-0021: expected=disqualify actual=manual_review
  LEAD-0026: expected=qualify actual=manual_review
  LEAD-0027: expected=qualify actual=escalate
  LEAD-0030: expected=qualify actual=escalate
  LEAD-0033: expected=manual_review actual=disqualify
  LEAD-0036: expected=qualify actual=manual_review
  LEAD-0037: expected=manual_review actual=escalate
  LEAD-0040: expected=qualify actual=manual_review
  LEAD-0041: expected=manual_review actual=escalate
  LEAD-0042: expected=qualify actual=manual_review
  LEAD-0046: expected=qualify actual=manual_review
  LEAD-0049: expected=qualify actual=manual_review
  LEAD-0055: expected=hold actual=manual_review
  LEAD-0061: expected=hold actual=manual_review
  LEAD-0063: expected=hold actual=manual_review
  LEAD-0070: expected=hold actual=manual_review
[Eval] RESULT: FAIL
```

**Agreement: 58/75 — FAIL** against the keyed gate (≥ 69/75), and below the
68/75 fallback baseline it was required to strictly beat. Reported as-is;
the gate was fixed before this run per the gate-before-code rule in
`eval_config.py`, and it is not adjusted after seeing the result.

**Every row used the LLM layer as intended** — `context_was_used=1` and
`used_fallback=0` on all 75 rows, `fallback_reason` empty throughout
(verified: `SELECT COUNT(*) FROM decisions WHERE context_was_used=1 AND
used_fallback=0` = 76 across this run plus one smoke-test row). The FAIL is
a genuine model-quality result, not a fallback contaminating the sample.

**Miss pattern.** The 17 misses cluster into two failure modes, not random
noise:
- **Under-committing on clear-cut cases (8 misses):** 6× `qualify` →
  `manual_review`, 2× `qualify` → `escalate`. The LLM routes several
  high-confidence, clearly-qualifying leads to a review queue instead of
  qualifying them outright — more conservative than the ground truth.
- **Avoiding `hold` (4 misses):** 4× `hold` → `manual_review`. This is the
  *same* category the deterministic fallback also cannot produce (it has
  no rule that emits `hold`) — the LLM has `hold` available in its schema
  and system prompt but converges on `manual_review` instead. Grounding in
  retrieved context did not close this specific gap.
- Remaining 5 misses are scattered: `manual_review` → `escalate` (×2),
  `manual_review` → `hold`/`disqualify` (×2), `disqualify` → `manual_review` (×1).

**Interpretation, stated plainly:** the recorded outcome bullet in the
README says decisions are "grounded in retrieved precedent when the LLM
layer is active" — that is true (every row here cites specific retrieved
document IDs in its reasoning, see EXAMPLE_OUTPUTS.md). It does not claim
the LLM path is *more accurate* than the deterministic fallback against
this synthetic ground truth, and on this measurement it is not — 58/75 vs
68/75. Grounding and accuracy are different claims; this repo can now
support the first with evidence and cannot support the second.

## Honest limitations (keyed run)

- **Same synthetic-data and small-n caveats as the fallback run** — 58/75
  is a point estimate against generated labels, not a real-world accuracy
  claim.
- **Single run, no retries or prompt tuning.** This is the first and only
  keyed run recorded. No prompt iteration was attempted to improve the
  score — doing so after seeing this result would be gate-shopping, which
  the gate-before-code rule above explicitly rules out. If the system
  prompt or model is revised later, that is a new run with its own entry
  here, not a silent re-run.
- **One compare run, one model.** `claude-sonnet-4-6` was used throughout
  (the originally hardcoded `claude-sonnet-4-20250514` had retired); no
  comparison against other models was attempted.

The keyed with/without-context compare run referenced in the README Outcome section is committed verbatim as Example 8 in [EXAMPLE_OUTPUTS.md](EXAMPLE_OUTPUTS.md).

## Reproduce

```bash
# Fallback mode
cp .env.example .env   # leave ANTHROPIC_API_KEY empty
uvicorn app:app --port 8000
python run_eval.py

# Keyed mode
cp .env.example .env   # then set ANTHROPIC_API_KEY to a real key
uvicorn app:app --port 8000
python run_eval.py
```
